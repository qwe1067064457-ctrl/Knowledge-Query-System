# Docs README

`docs/` 是当前仓库的轻量正式文档区。

现阶段项目的文档分工是：

- `notes/`：主知识库
- `CONTEXT.md`：AI / 新对话入口
- `README.md`：人类入口
- `docs/`：相对稳定、相对正式的文档

## 适合放这里的内容

- 已经比 working notes 更稳定的设计文档
- 需要长期引用的架构决策记录
- 你希望明确视为“当前正式材料”的文档

## 不建议第一时间放这里的内容

- 还在快速变化的临时方案
- 深过程、强探索性的笔记
- 还没复核过的 AI 中间产物

这些内容更适合先放：

- `notes/working/`
- 对应主题下的 `notes/` 目录

## 当前内容

- `knowledge-hybrid-retrieval-ai-plan.md`
  - skill-first + hybrid retrieval 的较正式方案文档
- `adr/`
  - 架构决策记录与 ADR 说明

## 建议阅读顺序

1. `../README.md`
2. `../CONTEXT.md`
3. `../notes/README.md`
4. 再进入具体的 `docs/` 文档
