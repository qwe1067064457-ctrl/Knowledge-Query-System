# 📚 Knowledge System Architecture (For CodeAgent)

## 🧠 Overview

本文件定义**知识库系统的总体架构（High-Level Design）**。

* 当前仅描述**模块划分与核心流程**
* **不包含具体实现细节**
* 每个模块将在后续以独立文件补充（`/modules/*.md`）

> ⚠️ CodeAgent 使用约定：
>
> * 本文件用于理解系统全局结构
> * 不直接生成代码
> * 所有实现必须参考子模块说明文件

---

## 🏗️ System Architecture

系统由以下核心模块组成：

```
User Input
   ↓
Intent Recognition
   ↓
┌───────────────┬───────────────┬───────────────┐
│ Knowledge QA  │ File Agent    │ Normal Chat   │
└───────────────┴───────────────┴───────────────┘
   ↓
Context Manager
   ↓
Retrieval System
   ↓
LLM Generation
   ↓
Response
```

---

## 1️⃣ Data Layer（数据层）

### 🎯 目标

支持多领域（法律 / 医学 / 未来扩展）+ 多租户 + 权限隔离

### 🧩 设计原则

#### 1. 数据隔离（必须实现）

* **物理隔离**

  * 不同 domain 使用不同 collection / 文件夹
* **逻辑隔离**

  * 使用 `group_id` / `namespace`
* **权限隔离**

  * 基于 `access_control`（RBAC）

---

### 📂 文档处理策略

| 类型       | 处理方式     | 输出格式      |
| -------- | -------- | --------- |
| PDF      | OCR + 解析 | Markdown  |
| DOC/DOCX | 结构解析     | Markdown  |
| HTML     | 清洗提取     | Markdown  |
| Excel    | 数据建模     | JSON / DB |

---

### 📌 统一约定

```yaml
input: 原始文件
output:
  - markdown文本
  - metadata
  - chunks
```

---

## 2️⃣ Intent Recognition（意图识别）

### 🎯 目标

识别用户意图并路由到对应模块

### 🧩 分类

* `knowledge_qa` → 知识问答
* `file_operation` → 文件操作
* `normal_chat` → 普通对话
* `uncertain` → 需澄清

---

### ⚙️ 策略

```
规则优先 → 语义分类 → fallback
```

---

### 📌 输出格式

```json
{
  "intent": "knowledge_qa",
  "confidence": 0.92,
  "domain": "legal"
}
```

### 补充
```
后期将使用微调小模型作为意图识别大脑
```

## 🤖 SubAgent Architecture（子代理架构）

### 🎯 目标

将系统拆分为：

* **Main Agent（主Agent）** → 负责规划 / 路由 / 决策（低Token消耗）
* **Sub Agents（子Agent）** → 负责具体执行（高Token消耗）

实现：

* 降低整体 Token 成本
* 提升复杂任务可控性
* 支持多任务并行扩展
