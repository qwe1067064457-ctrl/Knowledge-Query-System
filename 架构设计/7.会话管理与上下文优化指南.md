# 🧠 Claude Code 会话管理与上下文优化指南

**来源**：Anthropic 官方博客 (2026-04-15)
**核心概念**：上下文窗口 (Context Window)、上下文腐烂 (Context Rot)、压缩 (Compaction)、子 Agent (Subagents)

---

## 1. 核心原理：上下文与性能

### 1.1 上下文窗口 (Context Window)
*   **定义**：模型在生成下一个响应时能“看到”的所有内容。
*   **包含**：系统提示词、对话历史、所有工具调用及其输出、读取的所有文件。
*   **容量**：Claude Code 拥有 100 万 token 的上下文窗口。

### 1.2 上下文腐烂 (Context Rot)
*   **现象**：随着上下文增长，模型性能会轻微下降。
*   **原因**：
    1.  注意力机制分散到更多 token 上。
    2.  旧的、无关的内容开始干扰当前任务。
*   **临界点**：当接近上下文窗口上限时，系统会自动将任务总结为较小的描述，并在新窗口中继续。**此时模型的智能水平最低。**

---

## 2. 五大决策分支 (Every Turn as a Branching Point)

当 Claude 完成一个指令后，你面临以下 5 种选择。CodeAgent 需根据当前状态自动判断执行路径：

| 选项 | 指令/操作 | 适用场景与逻辑 |
| :--- | :--- | :--- |
| **继续** | Send another message | **默认路径**。当任务连续且上下文负载仍相关时使用。 |
| **回溯** | `/rewind` 或 双击 Esc | **修正路径**。当模型走错路时，跳回错误发生前的节点，丢弃后续无效上下文，重新提示。 |
| **清除** | `/clear` | **新任务**。开始一个全新的任务，从零开始构建上下文。 |
| **压缩** | `/compact` | **减负**。当会话过长时，让模型总结历史并替换为摘要，以减少上下文负载。 |
| **子 Agent** | Agent Tool | **并行/隔离**。将产生大量中间输出的子任务交给拥有独立上下文的子 Agent 处理。 |

---

## 3. 关键策略详解

### 3.1 新建会话 vs. 持续会话
*   **原则**：开始新任务时，通常应开启新会话。
*   **例外**：对于高度相关的连续任务（如刚实现的功能紧接着写文档），复用会话可以避免重新读取文件，节省成本和时间。

### 3.2 回溯 (Rewinding) 而非纠正
*   **策略**：不要在错误的尝试后直接说“错了，重试”。这会增加上下文腐烂。
*   **操作**：
    1.  使用 `/rewind` 跳回文件读取后的状态。
    2.  重新提示：“不要使用方法 A，直接使用方法 B。”
*   **高级用法**：使用 `/rewind` 进行总结，让未来的自己给过去的自己写便条。

### 3.3 压缩 (Compaction) 的艺术
*   **主动压缩**：不要等到系统自动压缩（此时模型最笨）。在 100 万 token 用完前，主动使用 `/compact`。
*   **/compact vs /clear**：
    *   `/compact`：模型自动总结。**优点**：省事，可能保留隐藏细节；**缺点**：有信息丢失风险。
    *   `/clear`：**你**手动写下关键点（约束、文件、已排除方案），然后开始新会话。**优点**：完全可控，无腐烂。
*   **引导压缩**：使用指令如 `/compact focus on X, drop Y` 来引导模型保留重要内容。

### 3.4 子 Agent (Subagents)
*   **触发条件**：当一个子任务会产生大量中间输出，且这些输出在主任务中不再需要时。
*   **机制**：子 Agent 拥有独立的上下文窗口。它完成工作后，仅将最终结论返回给父会话。
*   **典型场景**：
    *   验证工作结果。
    *   阅读另一个代码库并总结实现方式。
    *   基于 Git 变更编写文档。

## 4. 决策辅助表 (Decision Matrix)

CodeAgent 在运行时应参考此表决定下一步动作：

| 当前情境 | 建议操作 | 原因 |
| :--- | :--- | :--- |
| **任务连续，上下文仍有用** | `Continue` | 避免重复构建上下文，节省 Token。 |
| **模型走错路，需要重试** | `Rewind` | 保留有用的文件读取，丢弃失败的尝试，利用已学知识重试。 |
| **会话过长，包含大量无用调试** | `/compact <hint>` | 低工作量的减负方式；可用提示词引导保留重点。 |
| **开启全新任务** | `/clear` | 彻底消除上下文腐烂；完全掌控携带的信息。 |
| **下一步会产生大量中间噪音** | `Subagent` | 噪音留在子进程；仅结论返回主进程。 |

---

## 5. 避坑指南 (Anti-patterns)

1.  **被动压缩**：依赖系统自动压缩。由于发生在上下文末尾，模型此时注意力最差，容易丢失关键信息（如调试过程中顺带提到的其他警告）。
2.  **过度堆叠**：在同一个会话中处理完全无关的任务，导致上下文臃肿且混乱。
3.  **忽视回溯**：试图在错误的上下文中通过文字纠正模型，导致模型陷入“解释自己错误”的循环，而非重新执行。
