#clau 📊 RAG智能问答系统流程（文字版）

## 🧠 总体流程概览

系统整体链路如下：

```
用户输入
 → 会话管理
 → 意图理解（含Query改写）
 → 检索引擎
 → 生成服务（LLM）
 → 输出服务（流式返回）
```

---

## 1️⃣ 会话管理（Session Layer）

### 🎯 职责

* 管理用户会话生命周期
* 提供上下文信息

### 🔄 流程

```text
用户提问
 → 会话初始化（taskId, conversationId）
 → 记忆加载（历史 + 当前）
 → 请求转发
```

---

## 2️⃣ 意图理解层（Intent & Query Layer）

### 🎯 核心能力

#### 1. Query 改写

* 对用户问题进行：

  * 重写
  * 拆分
  * 规范化

#### 2. 意图识别

* 判断用户意图：

  * 知识问答
  * 普通对话
  * 其他操作

#### 3. 意图引导

* 当信息不足时：

  * 返回引导问题
  * 请求补充信息

---

### ⚠️ 分支逻辑

```text
if 信息不完整:
    返回引导提示（不进入检索）
else:
    进入检索阶段
```

---

## 3️⃣ 检索引擎（Retrieval Layer）

### 🎯 职责

提供高质量上下文证据

---

### 🔄 检索流程

```text
检索请求
 → 意图召回检索
 → 全局向量检索
 → MCP工具调用（可选）
 → 去重 / 重排序
 → 返回检索结果
```

---

### 📌 关键能力

* 多路召回（Hybrid Retrieval）
* 工具增强（MCP）
* 结果优化：

  * 去重
  * rerank

---

## 4️⃣ 生成服务（LLM Layer）

### 🎯 职责

基于上下文生成最终答案

---

### 🔄 处理流程

```text
构建 Prompt：
(system + context + history + user)

 → 调用模型
 → 首包返回（约60s内）
```

---

### 🔁 重试机制

```text
loop:
    调用模型
    if 失败:
        重试（最多1次）
    else:
        返回结果
```

---

## 5️⃣ 输出服务（Response Layer）

### 🎯 职责

对结果进行处理并返回用户

---

### 🔄 流程

```text
生成结果
 → 流式输出（SSE）
 → 消息结构化
 → 标题生成
 → Trace记录
 → 返回用户
```

---

## 6️⃣ 并行处理（Parallel）

### 🎯 特点

系统支持并行执行：

```text
生成阶段 ↔ 输出阶段
```

* 边生成边返回（流式）
* 降低用户等待时间

---

## 7️⃣ 系统关键机制总结

### ✅ 1. 分层解耦

| 层级  | 职责   |
| --- | ---- |
| 会话层 | 上下文  |
| 意图层 | 理解问题 |
| 检索层 | 找证据  |
| 生成层 | 生成答案 |
| 输出层 | 返回结果 |

---

### ✅ 2. 容错机制

* LLM失败自动重试
* 信息不足提前终止
* 检索增强保证稳定性

---

### ✅ 3. 性能优化

* 流式输出（SSE）
* 并行处理
* 首包时间控制（60s）
